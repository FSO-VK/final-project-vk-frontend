# Сегмент shared/api

В этом сегменте реализуется API бэкенда:

* `./client` содержит реализацию http клиента в соответствии
  с протоколом front-back
* `./endpoints/` содержит ручки для доступа к бэкенду.
  Для каждой ручки определяются DTO в рамках соответствующего домена.

Для
каждого DTO создается zod-схема и только для DTO, для
бизнес-сущностей она не создается. То есть библиотека zod не
должна покидать пределы shared/api. Валидация входных данных для
ручек не нужна, так как они приходят либо из моделей сущностей,
который заведомо корректны, или из DTO форм, которые, в свою
очередь, также валидируются.
